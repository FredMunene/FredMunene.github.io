Markdown is enough for most engineering notes.

## What I tend to write

- System notes
- Release notes
- Short explainers
- Implementation snippets

## Example

I often keep examples very short:

```bash
curl -L https://gitongacodes.co.ke/blog/launching-the-blog.md
```

That keeps posts easy to scan and easy to maintain.
